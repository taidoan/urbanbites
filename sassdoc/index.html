    mq

    @mixin mq($value, $type: min, $media-feature: width, $orientation: null) { ... }

    Description

    Media query mixin to handle various breakpoint units and types.

    This mixin supports breakpoints defined in em, rem, px, or as keys in a predefined breakpoints map. It can create min-width or max-width or even min-height or max-height queries and allows for additional media feature flexibility, including orientation.

    Parameters

    parameter Nameparameter Descriptionparameter Typeparameter Default value
    $value

    The breakpoint value or key from the map.

    		Number or String— none
    $type

    The type of media query, either min or max.

    		Stringmin
    $media-feature

    The media feature to use.

    		Stringwidth
    $orientation

    Optional. The device orientation (portrait or landscape).

    		Stringnull

    Example

    // Using a named key from the breakpoints map, defaulting to min-width
@include mq('medium') {
  .example { color: blue; }
}

// Explicit max-width query
@include mq('medium', 'max') {
  .example { color: red; }
}

// Including an orientation condition
@include mq('medium', $orientation: 'landscape') {
  .example { color: green; }
}

    Throws

    • Invalid or missing breakpoint value provided.

    Requires

    • [function] em
    • [function] is-em
    • [function] is-rem
    • [function] is-px
    • [function] em
    • [variable] breakpoints-map
    • [variable] breakpoints-map
